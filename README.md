# FB Job Poster

Автопостинг и планирование вакансий на твои Facebook-страницы (8 штук)
через официальный Graph API. Без обходов и спама — только страницы,
которыми ты владеешь.

## Что умеет
- Запостить один текст/картинку сразу на все страницы
- Запланировать пост на конкретное время (нативно в Facebook)
- Повторять публикацию по расписанию (напр. пн+чт в 10:00)
- Проверить валидность всех токенов одной командой

## Установка
```bash
cd fb-job-poster
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
cp config.example.json config.json   # затем впиши свои токены
```

## Настройка приложения (один раз)

1. Зайди на <https://developers.facebook.com/> → **создай приложение** (тип
   *Business*).
2. **Settings → Basic**: скопируй **App ID** и **App Secret** →
   впиши в `.env` (скопируй из `.env.example`).
3. Добавь продукт **Facebook Login** → в его настройках, в поле
   **Valid OAuth Redirect URIs**, впиши:
   ```
   http://localhost:8000/
   ```
   (для `localhost` Facebook разрешает http — это исключение).
4. Приложение можно оставить в режиме **Development** — для своих страниц,
   где ты админ, права работают без App Review. Убедись, что твой аккаунт
   указан в **App Roles** как Admin/Developer.

## Вход через браузер и авто-сбор токенов

```bash
cp .env.example .env      # впиши FB_APP_ID и FB_APP_SECRET
python login.py
```

- Откроется браузер на странице Facebook → залогинься и нажми **«Разрешить»**.
- Скрипт сам обменяет код на долгоживущий токен, вытянет **токены всех
  твоих страниц** и запишет `config.json`.
- Page-токены, полученные так, обычно **не истекают**, пока не сменишь
  пароль/права.

> Если страниц не видно — проверь, что ты их **админ** и что на шаге
> согласия отметил доступ к нужным страницам.

### Проверь, что всё ок
```bash
python cli.py check
```
Должно вывести `[OK] <имя страницы>: ...` по каждой из 8.

## Десктопное приложение (рекомендуется)

Нативное окно на компьютере (без браузера) с тем же интерфейсом: композер
с живым превью «как в Facebook», выбор страниц, предпросмотр (dry-run),
расписание и анти-спам.

Запуск из исходников:
```bash
pip install -r requirements-desktop.txt
python desktop.py
```

Возможности окна:
- **«Подключить страницы»** — вход через браузер прямо из приложения;
  после «Разрешить» страницы появляются автоматически.
- **Живое превью** — баннер и финальный текст по каждой странице
  (подстановка `{page}` и эмодзи-вариация).
- **Предпросмотр** (без публикации) и **Опубликовать** (с паузами).
- Планирование через поле даты/времени.

### Сборка в .exe / .app (двойной клик)
```bash
pip install pyinstaller
python build_app.py
```
Готовый файл — в папке `dist/`:
- Windows → `dist/FB Job Poster.exe`
- macOS → `dist/FB Job Poster.app`
- Linux → `dist/FB Job Poster`

`config.json` и `.env` держи **рядом** с готовым файлом — они не вшиваются
внутрь (это и хорошо: токены остаются у тебя локально).

> Linux требует GTK/Qt-бэкенд для окна — см. комментарии в
> `requirements-desktop.txt`.

## Использование через CLI

Опубликовать сейчас на все страницы:
```bash
python cli.py post --text "Ищем повара в кафе. Зарплата ..., график ...
Пиши в личку или звони +380..." --link https://t.me/...
```

Запланировать (от 10 минут до 30 дней вперёд — ограничение Facebook):
```bash
python cli.py post --text "..." --at "2026-06-25 09:00"
```

### Тест без публикации (dry-run)
Посмотреть, что и куда уйдёт, **ничего не публикуя** — покажет баннер и
финальный текст по каждой странице:
```bash
python cli.py post --text "Ищем повара {page}!" --image https://.../banner.jpg --dry-run
```
Можно тестировать даже без токенов, на примере конфига:
```bash
python cli.py --config config.example.json post --text "..." --dry-run
```

### Анти-спам (включено по умолчанию)
Чтобы одинаковые посты на 8 страниц не попали под фильтр Facebook:
- **Вариация текста** — в конец добавляется ротируемый эмодзи (разный для
  каждой страницы). Плюс плейсхолдер `{page}` подставляет имя страницы.
- **Пауза между страницами** — случайная (по умолч. 2–6 сек).

Настройка/отключение:
```bash
python cli.py post --text "..." --no-vary             # без вариации текста
python cli.py post --text "..." --delay-min 5 --delay-max 15
```

Повторять напоминание (пока процесс запущен):
```bash
python cli.py recurring --text "Вакансия ещё открыта! ..." --days mon,thu --hour 10
```

С картинкой:
```bash
python cli.py post --text "..." --image https://example.com/vacancy.jpg
```

## Запуск планировщика в фоне (на сервере/Деке)
```bash
nohup python cli.py recurring --text "..." --days mon,thu --hour 10 \
    > poster.log 2>&1 &
```
Или оформи как systemd-сервис, если нужен автозапуск.

## Важно
- Только свои страницы. Массовый тегинг друзей не делаем — это спам и бан.
- Один и тот же текст на 8 страниц подряд может попасть под антиспам-фильтры.
  Лучше слегка варьировать текст и не постить всё в одну секунду.
- `config.json` с токенами **не коммить** (он в `.gitignore`).
